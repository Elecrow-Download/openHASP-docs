With this online font converter tool you can create an openHASP binary font from any TTF, OTF or WOFF font. You can select ranges of Unicode characters and speficy the bpp (bit-per-pixel).

The font converter is designed to be compatible with LVGL. The offline version of the converter is available here

How to use the font converter?

  1. Give name to the output font. E.g. "arial_40"
  2. Specify the height in pixels. E.g. 40
  3. Set the bpp (bits-per-pixel). Higher value results smoother (anti-aliased) font, but larger files.
  4. Select the TTF, OTF or WOFF font
  5. Set a range of Unicode characters to include in the binary font or list the characters in the Symbols field
  6. Optionally choose another font too and specify the ranges and/or symbols for it as well. The characters will be merged into the final font file.
  7. Click the Convert button to download the resulting binary font file.

How to use the generated fonts in openHASP?

  1. PSram is required to use binary fonts.
  2. Copy the resulting binary font file to the flash partition of your plate.
  3. In the jsonl code use "text_font": "arial_40" without the extention or use command p1b2.text_font=arial_40
You can use both "Range" and "Symbols" or only one of them
